## THE LOCKED WORKFLOW

This is the production contract. It does not change.

```
STEP 1 — NATHAN PROVIDES INPUT
Nathan gives Claude one of the following:

  A. CONTEXT     A situation, scenario, or real-world moment
  B. PRINCIPLE   A training truth, doctrine statement, or ETKM belief
  C. QUOTE       A direct quote — from Nathan, a student, or a source
  D. PACKAGE     A combination of the above, optionally including an image

No additional direction required. Claude reads the input,
identifies the message, and builds the full package.

─────────────────────────────────────────────────────────

STEP 2 — CLAUDE BUILDS THREE PLATFORM PACKAGES

Claude delivers one complete package per platform:

  FACEBOOK PACKAGE
  ├── Canva brief (graphic type, canvas size, exact copy, image brief)
  ├── Graphic copy locked (headline, supporting line, footer)
  └── Facebook caption (community tone, CTA, hashtags)

  INSTAGRAM PACKAGE
  ├── Canva brief (graphic type, canvas size, exact copy, image brief)
  ├── Graphic copy locked (headline, supporting line, footer)
  └── Instagram caption (hook line first, punchy, hashtags)

  LINKEDIN PACKAGE
  ├── Canva brief (graphic type, canvas size, exact copy, image brief)
  ├── Graphic copy locked (headline, supporting line, footer)
  └── LinkedIn caption (practitioner voice, white space, soft CTA)

Each platform package is built for that platform specifically.
Copy is never pasted across platforms unchanged.
Graphics may share a design but captions always differ.

─────────────────────────────────────────────────────────

STEP 3 — CANVA BUILDS FROM THE BRIEF
Canva receives the Canva brief. Builds exactly what it specifies.
No copy decisions. No design improvisation. Brief is the authority.

─────────────────────────────────────────────────────────

STEP 4 — QC BEFORE POSTING
All 6 QC gates run before any graphic is approved.
Any failure returns to the responsible step — not back to zero.
```

## THE THREE GRAPHIC TYPES

Every ETKM social graphic is one of three types.
Declare the type before writing copy or building in Canva.

---

### TYPE A — STATEMENT + IMAGE

**What it is:** B&W cinematic photo background. Oversized bold text dominates
the lower 60–70% of the frame. One message. High contrast. Unmissable.

**When to use:**
- TOFU awareness content — stopping the scroll cold
- Messages about the real world (danger, awareness, readiness)
- When a real-world scene amplifies the message
- Highest impact format — use for strongest statements

**Benchmarks:** B1 (Emergencies), B5 (Be Alert Be Aware)

**Design rules:**
- Photo is always B&W — no color photos ever
- Photo subject faces away or in profile — never direct camera stare
- Photo communicates vulnerability OR awareness — never violence in progress
- Text sits in the lower 50–70% of the frame
- Background photo is naturally dark in text zone — no gradient needed
- One word or short phrase in RED — top of text block only
- Remaining text in WHITE bold condensed
- ETKM logo bottom right — always white circle badge
- No URL on Type A unless event-specific

**Photo direction (when sourcing):**
Load etkm-cinematic-doctrine. Select scene intent that matches message.
Behavioral states: Baseline Aware or Heightened only.
Never: mid-fight, strike in progress, aggressor perspective.

---

### TYPE B — TYPE ONLY (DARK)

**What it is:** Pure black or dark textured background. Typography is the
entire design. No photo. Clean, bold, unapologetic.

**When to use:**
- Principle-based statements and doctrinal truths
- Quote-style content
- When the message is so strong it needs no visual support
- TOFU and MOFU content
- Fastest to produce — highest volume use

**Benchmarks:** B2 (Stress Doesn't Ask), B4 (You Don't Fail Under Pressure)

**Design rules:**
- Background: pure black (#000000) or dark textured (charcoal/concrete grain)
- Optional: very subtle texture overlay — never distracting
- Small red category label at top left — 8–10px, uppercase, spaced
  (Examples: STRESS / AWARENESS / PREPARED / PROTECTION / MINDSET)
- Main headline: white, bold condensed, left-aligned
- Supporting line: red, all caps, condensed — one line maximum
  OR white, smaller weight — depends on message hierarchy
- Red horizontal rule as visual separator between statement and footer
  (use when event/URL info follows)
- URL bottom center: ETXKRAVMAGA.COM — always present on organic posts
- ETKM logo bottom right — always
- No images. No decorative elements. The type IS the design.

---

### TYPE C — SPLIT LAYOUT

**What it is:** B&W photo occupies left panel. Black right panel carries
structured information. Used for people, events, announcements.

**When to use:**
- Guest instructor announcements
- Event promotions
- Student spotlights
- Instructor profiles
- Any content where a face and facts need to coexist

**Benchmark:** B3 (CBLTAC John Wilson)

**Design rules:**
- Photo panel: left side, 40–45% of frame width, B&W only
- Info panel: right side, 55–60% of frame width, pure black background
- Small red label at top of info panel (event code, category, series name)
- Subject name or headline: white, large, bold — 2–3 lines maximum
- Descriptor text: gray or light gray, smaller weight
- Key facts: white, bold — short lines, one fact per line
- Red horizontal rule separating facts from footer
- Footer bar (full width, very bottom): small white text — date, location, URL
- ETKM logo bottom right of info panel
- Photo: cinematic, directional light, subject facing slightly toward info panel

## PLATFORM SPECIFICATIONS

Every graphic is built to exact platform specifications.
Declare platform BEFORE declaring canvas size.

---

### FACEBOOK

| Format | Canvas Size | Ratio | Primary Use |
|--------|------------|-------|-------------|
| Square Post | 1080 × 1080 px | 1:1 | Feed post — universal |
| Landscape Post | 1200 × 630 px | 1.91:1 | Link share, event graphics |
| Story | 1080 × 1920 px | 9:16 | Temporary content, reels cross-post |
| Cover Photo | 851 × 315 px | 2.7:1 | Profile cover — not campaign use |

**Facebook communication style:**
- Slightly longer captions acceptable — up to 3 short paragraphs
- Community tone — speaking to the Tyler, TX audience
- Direct. Not corporate.
- End with a single clear CTA: "Schedule your free trial →" or "Link in bio."
- Event posts: include date, location, URL in caption always
- Hashtags: 3–5 maximum, relevant only

**Facebook graphic type priority:**
Type A (Square) for awareness content
Type C (Landscape) for events and announcements
Type B for principle/doctrine content

---

### INSTAGRAM

| Format | Canvas Size | Ratio | Primary Use |
|--------|------------|-------|-------------|
| Square Post | 1080 × 1080 px | 1:1 | Standard feed post |
| Portrait Post | 1080 × 1350 px | 4:5 | Fills more feed real estate |
| Story / Reel Cover | 1080 × 1920 px | 9:16 | Stories, Reel thumbnails |
| Carousel Slide | 1080 × 1080 px | 1:1 | Multi-slide posts |

**Instagram communication style:**
- Hook in first line — must stand alone before "more" cutoff
- First line is the scroll-stopper. Make it earn the tap.
- Body: 3–5 short lines maximum for awareness content
- Emojis: none — ETKM voice does not use emojis
- Hashtags: 5–10, end of caption, not mid-copy
- Always end with one action: "Free trial → link in bio"
- Stories: text overlay minimal — image/video speaks first

**Instagram graphic type priority:**
Type A (Portrait 4:5) for maximum feed presence
Type B (Portrait or Square) for doctrine/principle posts
Carousel: Type B slides with Type A cover — see Carousel System below

---

### LINKEDIN

| Format | Canvas Size | Ratio | Primary Use |
|--------|------------|-------|-------------|
| Square Post | 1200 × 1200 px | 1:1 | Standard feed post |
| Landscape Post | 1200 × 627 px | 1.91:1 | Article headers, link previews |
| Portrait Post | 1080 × 1350 px | 4:5 | Feed post — high presence |
| Document/Carousel | 1080 × 1080 px | 1:1 | PDF carousel (native format) |

**LinkedIn communication style:**
- Professional but not corporate — Nathan is a practitioner, not a consultant
- Leads with a principle, observation, or insight — not a sales message
- Short paragraphs — 1–2 sentences each
- White space between every paragraph — LinkedIn compresses text visually
- No hashtag walls — 3 max, end of post only
- CTA is softer: "Worth thinking about." or "What's your approach to this?"
  or a link to the relevant resource
- Audience: professionals, law enforcement, trainers, executives, veteran community
- Tone: peer-to-peer authority. Nathan speaks as a practitioner to practitioners.
- Type C graphics perform best here — professional, structured, credible

**LinkedIn graphic type priority:**
Type C for people and event content
Type B for principle/insight content — cleaner reads in professional context
Type A — use selectively, ensure photo is workplace-appropriate in tone

## CAROUSEL SYSTEM

Carousels are multi-slide posts. Instagram and LinkedIn native.
Facebook supports them but carousel performs lower there.

**Carousel structure — 5 to 7 slides:**

```
SLIDE 1 — COVER (Hook)
Type: Type A or Type B
Job: Stop the scroll. Deliver the promise of what's inside.
Text: Bold statement or question. "Swipe to see all 5."
Copy formula: "[Number] [things / reasons / signs / rules] that [outcome]"

SLIDES 2–5 (or 2–6) — CONTENT SLIDES
Type: Type B (consistent — dark background, clean)
Job: One point per slide. Each slide stands alone.
Layout: Red category label → Bold white point → 1–2 sentence explanation
Slide number (optional): Small, bottom left, red — "02 / 05"

FINAL SLIDE — CTA
Type: Type B with CTA treatment
Job: Convert attention to action
Layout: Strong closing statement → Red rule → URL + Logo
Text: "Ready to go from [before state] to [after state]?"
      "Your first class is free." + etxkravmaga.com/free-trial
```

**Carousel design rules:**
- All slides same background (pure black preferred)
- Typography consistent across all slides
- Slide number indicator optional but recommended
- First slide must work as a standalone post if carousel isn't opened
- Last slide always has the CTA — never leave the carousel without one

**Carousel copy rules:**
- Each slide is one idea. One.
- No slide requires reading the previous slide to make sense
- Keep body text under 15 words per slide
- The statement on each slide must be true on its own

## COPY SYSTEM

### Headline Formula Bank

Every ETKM graphic headline follows one of these formulas.
Copy is written by Claude before Canva builds anything.

**Truth Statement** (most common — Type A and B)
State a reality the audience already feels but hasn't articulated.
"Emergencies don't announce themselves."
"Stress doesn't ask for permission."
"Most people don't notice until it's too late."

**Reframe** (MOFU — challenging a belief)
"You don't fail under pressure because you weren't prepared."
"Confidence isn't the same as being ready."
"Fear isn't the problem. Surprise is."

**Imperative Sequence** (Type A — B5 pattern)
Three short imperatives that build on each other.
"BE ALERT. BE AWARE. BE PREPARED."
"RECOGNIZE. DECIDE. ACT."
"SEE IT. OWN IT. HANDLE IT."

**Question** (carousel covers, Instagram hooks)
"What would you do if you had three seconds to decide?"
"Do you know the difference between fear and danger?"
"When did you last practice staying calm under pressure?"

**Category Label + Statement** (Type B — B4 pattern)
Small red label (topic) + bold white statement
STRESS / "You don't fail under pressure because you weren't prepared."
AWARENESS / "The most dangerous moment is the one you didn't see coming."
PROTECTION / "Your job isn't to fight. It's to go home."

---

### Supporting Copy Rules

The supporting line is one sentence. Maximum.
It answers one of two questions:
- What does that mean? (clarify the headline)
- What do I do about it? (give the action or the consequence)

**Examples from benchmarks:**
"It shows up on its schedule — train on yours." (answers: what do I do)
"They fail because stress strips away access to what they know." (answers: what does that mean)

**Never:**
- Two sentences in the supporting line
- Restating the headline in different words
- Selling in the supporting line
- Hedging ("it's possible that..." / "many people find...")

---

### Caption Copy by Platform

Every graphic gets platform-specific caption copy.
The graphic and the caption are one package — not separate.

**FACEBOOK caption structure:**
```
[Hook line — same energy as the graphic headline]

[2–3 sentences expanding the idea — one per line]

[Bridge to ETKM — one sentence, never pushy]

[CTA — single, specific]
[URL if relevant]
```

**INSTAGRAM caption structure:**
```
[First line — must work as the only line seen. No context assumed.]
.
[2–4 short lines — punchy, direct, one idea each]
.
[CTA — one line]
[Hashtags — 5–8, end only]
```

**LINKEDIN caption structure:**
```
[Opening observation or principle — 1–2 sentences]

[Expansion — 2–3 short paragraphs, 1–2 sentences each]

[Personal/practitioner note — Nathan's perspective]

[Question or soft CTA — invites engagement]
[3 hashtags maximum]
```

## QC GATES FOR SOCIAL GRAPHICS

Run before every graphic is approved for posting.

**GATE 1 — Message Clarity**
- [ ] Can you read the headline in under 2 seconds at thumb-scroll speed?
- [ ] Does the headline communicate one clear message?
- [ ] Does the supporting line answer "what does that mean?" or "what do I do?"

**GATE 2 — Brand Standards**
- [ ] B&W photo only (no color photography)
- [ ] Black or dark background on Type B
- [ ] Red used once per graphic — not twice
- [ ] No gradients anywhere
- [ ] No colors outside: #000000, #FFFFFF, #CC0000, #575757, #BBBBBB
- [ ] ETKM logo present, bottom right, white circle badge

**GATE 3 — Platform Spec**
- [ ] Canvas size matches declared platform exactly
- [ ] Text stays within safe zone (no text within 80px of edge)
- [ ] Logo not clipped at edge
- [ ] Mobile legibility confirmed (headline readable at 375px screen width)

**GATE 4 — Copy Standards**
- [ ] No prohibited words: mastery, dominate, destroy, killer, beast, crush, elite, warrior
- [ ] No specific year count for Nathan's experience
- [ ] URL correct: etxkravmaga.com (never easttxkravmaga.com)
- [ ] Caption written for declared platform — not copy-pasted across all three
- [ ] Hashtags platform-appropriate

**GATE 5 — Cinematic Doctrine (Type A and C only)**
- [ ] Photo subject faces away or in profile — not direct camera stare
- [ ] No active violence in progress — awareness moment only
- [ ] Behavioral state matches scene intent
- [ ] Photo communicates vulnerability OR awareness — not aggression

**GATE 6 — North Star**
- [ ] Would this stop Nathan's scroll?
- [ ] Does this look like it belongs in a Nike campaign?
- [ ] Is this unmistakably ETKM — not generic self-defense content?

**Any gate failure = return to the relevant agent. Fix before posting.**

## WHAT THIS SYSTEM NEVER DOES

- Posts a graphic without locked copy from Claude first
- Uses color photography
- Uses gradients or drop shadows as design elements
- Puts red in more than one place per graphic
- Writes copy in Canva — copy is always locked before Canva opens
- Copies the same caption to all three platforms
- Shows violence in progress (mid-fight, strike landing)
- Uses the ETKM logo anywhere except bottom right
- Posts without running all 6 QC gates

## BATCH PRODUCTION PROTOCOL

When producing a content batch (weekly, monthly, campaign):

1. **Nathan declares:** Number of posts, platforms, topics or theme
2. **Claude produces:** Content strategy — which graphic types, which messages, which funnel stages
3. **Claude writes:** All copy for all posts — headlines, supporting lines, captions per platform
4. **Claude delivers:** One Canva brief per graphic — complete, no ambiguity
5. **Canva builds:** All graphics against the briefs
6. **QC runs:** All 6 gates on every graphic before any post is scheduled
7. **Nathan schedules:** Approved graphics via scheduling tool

**Minimum batch:** 4 posts (1 week of content — 4 platforms × 1 format each)
**Recommended batch:** 12 posts (3 weeks of content — allows scheduling buffer)
**Campaign batch:** As defined by campaign brief — see etkm-event-planning
