## SECTION 1 — PLATFORM ASSIGNMENTS (LOCKED)

### The Three-Tool Rule

| Tool | Assigned Lane | Non-Negotiable |
|---|---|---|
| **PowerPoint (M365)** | System of record — sales decks, curriculum, video export, lobby TV kiosk, live presentations, consult decks | The only tool with Morph + Speaker Coach + kiosk mode + hyperlink nav + MP4 export + Slide Master |
| **Canva** | Brand-speed output — carousels, lobby graphics, event flyers, social, testimonials | Brand Kit + Brand Controls enforce font/color lockdown for non-designers |
| **Google Slides** | Web publishing + collaboration — parent-night embeds, partner decks, multi-author SOPs | "Publish to web" iframe = free embeddable digital signage; best real-time co-edit |

**Dependency chain:** Canva exports PNG/JPG assets → imports into PowerPoint as images. Google Slides handles web-facing layer. PowerPoint is primary.

### When to Use Which Platform

**Use PowerPoint when:**
- The deck will be presented live (any context)
- The deck needs hyperlink navigation (consult menu)
- The output needs Morph transitions
- The output is a video (MP4 export)
- The lobby TV kiosk will run it
- Complex animations or triggers are required
- Speaker Coach rehearsal is planned

**Use Canva when:**
- Producing social carousels or single-post graphics
- Non-designer staff needs to stay on-brand fast
- Creating event flyers, seminar graphics, testimonial slides
- Lobby graphics as supplemental visual assets (PNG export only)

**Use Google Slides when:**
- The deck needs to be embedded on the ETKM website via iframe
- Multiple people are co-authoring simultaneously
- The deck will be shared as a live URL that auto-updates
- Parent-night deck (iframe on website + shareable link)

## SECTION 2 — DESIGN RULES (NON-NEGOTIABLE)

### Core Principles

- **Rule of One:** One idea per slide. Split any slide with two competing ideas.
- **Live deck density:** Headline ≤8 words. Supporting text ≤25 words total per slide.
- **Slidedoc density:** Full paragraphs allowed — designed to be read alone.
- **Whitespace:** ≥30–40% of every slide is empty. This is what makes premium decks look premium.
- **Every element earns its place:** If removing it costs the audience nothing — remove it.
- **Grid alignment:** Use PowerPoint Guides at 1/3, 1/2, 2/3. Every element snaps to a grid point.

### ETKM Brand — Color System

| Swatch | Hex | Usage |
|---|---|---|
| ETKM Black | `#0A0A0A` | Primary background |
| ETKM White | `#FFFFFF` | Primary text on dark |
| ETKM Red | `#CC0000` | Headlines, rules, CTA buttons, section markers |
| ETKM Charcoal | `#2A2A2A` | Secondary backgrounds |
| ETKM Muted | `#B3B3B3` | Secondary text on dark |
| ETKM Gray | `#666666` | Secondary text on light |

> ⚠️ `#FF0000` PERMANENTLY RETIRED. `#CC0000` only. Always.

### Typography System

| Role | Font | Size | Weight |
|---|---|---|---|
| Display / Power Statement | Montserrat Black | 60–96pt | 900, ALL CAPS, tight tracking |
| Section Headline | Montserrat Black | 36–44pt | 900, Title Case |
| Body | Inter Regular | 18–24pt | 400 |
| Caption / Footer | Inter Regular | 12–14pt | 400, 60% opacity |
| Big Number Callout | Montserrat Black | 180–240pt | 900 |

### Imagery Rules

- Use real ETKM training photos — not stock.
- Convert to grayscale in PowerPoint: `Picture Format → Color → Saturation 0%`
- Full-bleed images: 60–80% black overlay when text appears on top
- Single red accent element (glove, rule, logo) against B&W photo reinforces brand

### The Two-Artifact Rule

Every important content set produces two files:

| Live Deck | Leave-Behind / Slidedoc |
|---|---|
| Visual, sparse, image-led | Denser, full-sentence titles |
| ≤8 word headline | Headline = the full takeaway |
| .pptx | PDF |
| Supports the speaker | Stands alone without the speaker |

**Workflow:** Build live deck → `Save As → [name]_handout` → expand titles to sentences → add body paragraphs → add executive summary page.

## SECTION 4 — STORYTELLING FRAMEWORKS

### StoryBrand (Spine of All Sales and Parent Decks)

- Character (hero) = the prospect / parent / student — NEVER ETKM
- Problem = External + Internal + Philosophical
- Guide = ETKM (empathy + authority via proof — never adjectives)
- Plan = 3 steps only
- CTA = Direct + Transitional on every deck
- Stakes = Real failure outcome — ethical, never manufactured

**Final check on every deck:**
- [ ] Student is the hero
- [ ] ETKM is the guide
- [ ] Speaks to internal problem first
- [ ] CTA is bold and direct
- [ ] Reader leaves feeling capable — not scared and small

### Duarte Sparkline

Oscillate between "What Is" (current reality) and "What Could Be" (transformed future) ≥3 times in any 10-min talk. Land on New Bliss — the vivid, specific picture of a year-from-now student.

### Problem → Stakes → Solution → Result

Shortest workable arc for a 5–10 min sales call. Always end with one specific CTA — never "learn more."

### Pyramid Principle (Corporate + Grant Decks)

Lead with the answer on slide 1. Action titles state the takeaway — not the topic. MECE supporting points. SCQA opening.

### 10/20/30 Rule (Kawasaki)

10 slides, 20 minutes, 30pt minimum font. Best for corporate proposals and partnership pitches.

## SECTION 5 — DELIVERY PROTOCOLS

### Context → Method Mapping

| Context | Method | Key Rule |
|---|---|---|
| One-on-one consult | Laptop, Presenter View, hyperlink menu deck | Deck is a prop — you run the meeting |
| Small group (5–15) | Projected, Presenter View, 36pt minimum text | Three verbal pause points; 3-layer backup plan |
| Seminar (20–100+) | Widescreen projection, wireless clicker, one statement per slide | Fade transitions only; slides are backdrop not content |
| Virtual (Zoom / Teams) | Share app window (not full screen); PowerPoint Live in Teams | Notes under webcam; email PDF within 90 min of call |
| Async / Lobby TV | Kiosk mode (.ppsx autoplay) or Google Slides published URL | Every slide completes its job in ≤7 seconds |
| Email | PDF, named properly, under 5MB | Never .pptx as attachment |

### Three Delivery Mistakes (Never)

1. Designing for the wrong audience
2. No redundancy plan (laptop → USB → phone PDF — always three layers)
3. Letting the deck drive the presenter

## SECTION 6 — LOBBY TV / DIGITAL SIGNAGE

### Setup (PowerPoint Kiosk Mode)

1. `Transitions → uncheck "On Mouse Click" → check "After [seconds]" → Apply To All`
2. `Slide Show → Set Up Slide Show → Browsed at a kiosk (full screen)`
3. Save as `.ppsx`
4. `Win + R → shell:startup` → drop .ppsx shortcut for auto-launch on boot

> ⚠️ Every slide MUST have an automatic timing or the loop hangs.

### Lobby TV Design Rules

- Resolution: 1920×1080 (16:9)
- Text minimum: 36pt readable, 60–120pt headlines
- Transitions: Fade only
- Animations: Off
- Audio: Off
- Slide duration: 7–10 seconds

### Content Rotation (12–24 slides)

Class schedule → featured technique → event promo → testimonial → belt promotion → safety stat → instructor spotlight → referral offer → program spotlight (Women/Kids/Corporate) → hours + contact → welcome message → CTA

## SECTION 7 — VIDEO EXPORT

### PowerPoint → MP4

`File → Export → Create a Video → Full HD (1920×1080) → Use Recorded Timings and Narrations`

| Output | Dimensions |
|---|---|
| YouTube | 1920×1080 |
| Reels / Shorts | 1080×1920 (set slide size first) |
| Lobby TV preload | 1920×1080 |

> ⚠️ PowerPoint exports ~30.30fps — not exactly 30fps. Run through HandBrake for professional editing projects.

### Reels Template Workflow

1. `Design → Slide Size → Custom → 10.83" × 19.27"`
2. Build 6–8 slides with Morph between each
3. Write narration in speaker notes
4. Record per slide
5. Export MP4 → 30-second Reel ready

## SECTION 8 — ADVANCED POWERPOINT FEATURES

### Morph

Same-named object on two consecutive slides → PowerPoint interpolates the change. Use `!!ObjectName` naming in Selection Pane to force-morph different shapes.

**ETKM uses:** Cinematic intros, Ken Burns photo zoom, technique flows, number countdowns, map zooms.

> ⚠️ Does not survive export to Slides or Canva.

### Speaker Coach

`Slide Show → Rehearse with Coach` — analyzes pace, filler words, repetitive language, reading off slides, pitch monotony. ETKM rule: every instructor does 3 runs before going live with any consult deck.

### Hyperlink Navigation

`Insert → Link → Place in This Document` — builds a menu-driven consult deck. Always add a "Home" button on the last slide of every section. Test only in Slide Show mode.

### Sections + Summary Zoom

`Home → Section → Add Section` for chapter organization. `Insert → Zoom → Summary Zoom` for a visual table-of-contents that zooms in/out of each section. Modern and cinematic — no animation work required.

### Slide Master Setup (ETKM Standard)

- Top thumbnail: 16:9 size, ETKM theme colors, Montserrat/Inter fonts, `#0A0A0A` background, logo bottom-right, slide number bottom-left
- Child layouts: Title Slide, Section Header, Title+Content, Two Content, Big Number, Testimonial, Closing/CTA
- Delete all unused layouts
- Save as `ETKM_Master_v[N].potx` + `ETKM.thmx`

## SECTION 9 — PRODUCTION WORKFLOW

### File Naming

```
YYYY-MM-DD_Category_Audience_Description_vNN.ext
```

Example: `2026-05-04_Sales_NewStudent_ConsultDeck_v03.pptx`

Rules: ISO date, underscores not spaces, 2-digit version, never overwrite (archive old versions).

### Folder Structure

```
/ETKM Marketing & Decks/
  /01_Templates/    ← Master.potx, Theme.thmx, Components.pptx
  /02_Sales/        ← Active prospects, closed, master templates
  /03_Curriculum/   ← Adults, Kids, Womens
  /04_Events/       ← Per-seminar subfolders
  /05_Internal/     ← Instructor training, staff onboarding
  /06_Lobby_TV/     ← Current loop + archive
  /07_Social/       ← Canva folder
  /08_Brand_Assets/ ← Logos, photography, fonts
  /_Archive/
```

### Handoff Protocol

1. Designer delivers .pptx + PDF in same folder
2. Designer runs Speaker Coach once + adds notes per slide
3. Designer records walkthrough MP4 (2–3 sentences per slide)
4. Instructor watches walkthrough → runs Speaker Coach on themselves
5. After live use → instructor adds Comments pane feedback
6. Feedback drives next version

## SECTION 10 — WHAT NOT TO DO

- **Do not use #FF0000** — `#CC0000` only, permanently
- **Do not author Canva .pptx exports as final presentation files** — font drift, no Morph, no kiosk mode
- **Do not present directly from Google Slides in high-stakes contexts** — no Speaker Coach, no Morph, no kiosk
- **Do not build content before the Slide Master is locked** — it will need to be rebuilt
- **Do not send .pptx as an email attachment** — always PDF
- **Do not run a consult deck linearly** — navigate by prospect's stated priorities
- **Do not build one file to serve as both live deck and leave-behind** — always produce both artifacts
- **Do not skip redundancy planning** — laptop → USB → phone PDF always
- **Do not add animations without a reason** — if you wouldn't pay for it in a film, cut it
- **Do not set lobby TV slides without automatic timings** — the loop will hang

## SECTION 11 — RECOVERY PROTOCOLS

### Deck Fails During Live Presentation

1. Open PDF backup immediately (pre-loaded on phone or USB)
2. Continue presenting from PDF — no apology, no pause
3. If no PDF: present from memory using section headers as guide
4. After the meeting: diagnose the failure, add a third backup layer

### Fonts Substitute on Export or New Machine

1. Identify substituted font via `Home → Replace Fonts`
2. Reinstall Montserrat Black and Inter on the machine
3. Replace substituted fonts back to ETKM standard
4. Enable `File → Options → Save → Embed fonts in file` going forward
5. Test on the presentation machine before every live event

### Lobby TV Loop Hangs (Freezes on One Slide)

1. Verify all slides have automatic timings set — `Transitions → slide timing`
2. Re-enable kiosk mode: `Slide Show → Set Up Slide Show → Browsed at a kiosk`
3. Re-save as .ppsx
4. Restart the kiosk PC

### Morph Breaks on Export to Another Tool

Expected behavior — Morph does not survive export to Google Slides or Canva. Keep the Morph-dependent master in PowerPoint. Produce a Fade-only version for any portable/cross-platform use.
